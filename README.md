## TDDプロセスとそのステップが重要である理由
<li>失敗するテストを作成してそれを確認する要件に対応するrelevantテストを作成し、失敗の説明を簡単に理解できることを確認しました。
<li>機能するソフトウェアがあることを確認するために、最小限のコードを記述して合格させる。
<li>リファクタリング、テストの安全性に裏打ちされており、操作が簡単な巧妙に作成されたコードがあることを確認します
